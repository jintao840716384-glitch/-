# 太玄宗：法术机制设定 V1.4（精简表格版）

> 本版只保留开发前规则骨架：少解释，多表格；系统语言和玩家显示分列；状态集中管理；技能只引用模块。

## 0. 文档定位

- 本文是开发前规则骨架，不是最终数值表、代码说明书或教程。
- 技能、状态、结界、被动效果统一按“规则链”组合。
- 状态细节集中在状态总表；技能表只写目标、执行、引用模块和状态。
- 系统底层优先看“数值变动 / 状态变动 / 目标选择 / 行动控制”；玩家显示再写成伤害、治疗、减速、增伤、续命等表现。
- 新增技能时，能引用已有模块就不新增规则；现有模块表达不了时，才新增通用模块。

## 1. 核心结构

| 对象 | 组成 | 用途 |
| --- | --- | --- |
| 技能 | 入口目标选择 + 执行器 + 规则链 + 消耗 / 冷却 | 主动释放的完整入口。 |
| 状态 | 挂载对象 + 显示标签 + 系统类型 + 计数器 + 触发器 + 规则链 + 移除 / 转化 | 挂在单位、阵营或战场上的持续规则。 |
| 规则链 | 触发时机 → 目标选择 → 执行器 → 效果 / 数值变动 → 计数器变化 → 后续触发 | 复杂技能由多条规则链连续调用。 |
| 计数器 | 当前值 / 上限 / 增加 / 减少 / 清空 / 覆盖 / 阈值 | 只记录数值，本身不自带效果。 |
| 系统类型 | 底层规则视角：气血变化、速度修正、死亡保护、行动跳过等 | 给机制拆解、AI 生成规则和后续代码实现参考。 |
| 显示标签 | 玩家 / UI 视角：伤害、治疗、减速、控制、续命、护盾等 | 给玩家理解、技能描述、状态图标和 AI 判断参考。 |

## 2. 系统语言与玩家显示

| 系统底层写法 | 玩家显示写法 | 例子 |
| --- | --- | --- |
| 气血数值减少 | 伤害 / 持续伤害 / 反弹伤害 | 燃烧、中毒、石球术、荆棘反弹。 |
| 气血数值增加 | 治疗 / 回血 / 吸血 | 横排治疗、水疗术、寄生转疗、续命回血。 |
| 速度数值降低 | 减速 | 寒霜、缠绕。 |
| 伤害倍率提高 | 增伤 | 气势每层提高体术 / 力道伤害。 |
| 抗性数值降低 | 减抗 | 金光刃、庚金神雷。 |
| 冷却数值增加 | 冷却干扰 | 水蚀使目标释放的主动技能冷却 +1。 |
| 阻止死亡确认 + 气血增加 | 续命 / 免死 | 气血归零时不倒下，回血后移除状态。 |

## 3.1 目标选择器

| 目标选择器 | 目标池 | 选择方式 / 备注 |
| --- | --- | --- |
| 自身 | 施法者 | 固定选择自己。 |
| 友方单体 | 友方存活单位 | 手动或自动选择，通常允许选自己。 |
| 敌方单体 | 敌方合法目标 | 遵守前后排、遮挡等基础规则。 |
| 友方横排 / 敌方横排 | 指定阵营一整排 | 影响该排所有合法存活目标。 |
| 友方全体 / 敌方全体 | 指定阵营全体 | 影响所有存活单位。 |
| 随机敌方 / 随机友方 | 指定阵营合法目标池 | 随机选择，可配合连续执行。 |
| 血量比例最低友方 | 友方存活单位 | 自动选择当前气血比例最低者。 |
| 扩散 | 主目标 + 相邻副目标 | 先选主目标，再按扩散参数找副目标。 |
| 十字扩散 | 主目标 + 十字范围 | 用于终极炎爆等范围攻击。 |
| 敌我双方合法随机 | 敌我双方合法目标池 | 混乱等目标改写效果使用。 |

## 3.2 执行器

| 执行器 | 说明 |
| --- | --- |
| 单次执行 | 规则链只执行一次。 |
| 固定次数连续执行 | 连续执行固定次数，例如 4 次、6 次。 |
| 随机次数连续执行 | 连续执行随机次数，例如 3～6 次。 |
| 每次重新选目标 | 每段执行都重新调用目标选择器，常用于弹射。 |
| 固定目标连续执行 | 第一次确定目标组，后续多段都打同一批目标。 |
| 触发后再次调用 | 前一条规则触发后一条规则，后续规则可重新选择目标。 |

## 3.3 触发时机

| 触发时机 | 常见用途 |
| --- | --- |
| 状态附加时 | 获得护盾、获得气势、附加结界计数。 |
| 自身回合开始 | 持续伤害、结界压制、部分回合类状态。 |
| 行动前 / 释放技能前 | 行动不能检查、混乱目标改写。 |
| 技能结算中 | 多段伤害、附加状态、读取计数器。 |
| 造成伤害后 | 寄生转疗、吸血类规则、伤害后追加效果。 |
| 受到伤害时 | 护盾承伤、荆棘反弹、受击触发。 |
| 释放技能后 | 水蚀冷却 +1、释放后反噬或增益。 |
| 护盾归零时 | 护盾状态移除、破盾反爆。 |
| 气血归零时 | 续命、死亡前保护。 |
| 计数器变化 / 达到阈值 | 寒霜 5 层转冻结、资源满层触发。 |
| 自身回合结束 | 持续时间减少、回合末移除。 |

## 3.4 基础效果类型

| 效果类型 | 系统本质 | 玩家表现 / 用途 | 常见参数 |
| --- | --- | --- | --- |
| 气血减少 | 改变目标气血数值；可能触发死亡检查。 | 伤害、持续伤害、反弹伤害、爆燃伤害。 | 属性、倍率、抗性、护盾、触发范围。 |
| 气血增加 | 改变目标气血数值；通常不超过气血上限。 | 治疗、回血、吸血、续命回血。 | 倍率、上限、目标选择、治疗量修正。 |
| 护盾变化 | 增加、减少、读取或清空护盾值。 | 获得护盾、护盾抵挡、盾爆。 | 护盾值、读取对象、是否消耗、上限。 |
| 法力 / 冷却变化 | 改变法力、技能冷却等数值。 | 耗蓝、回蓝、冷却干扰。 | 增加、减少、指定技能、触发条件。 |
| 数值修正 | 改变速度、抗性、伤害、治疗、消耗等公式参数。 | 减速、增伤、减抗、独立减伤。 | 修正对象、修正类型、数值、持续。 |
| 状态附加 / 移除 | 添加或移除状态实例。 | 上状态、净化、驱散。 | 状态名、层数、持续、命中、净化规则。 |
| 计数器操作 | 读取、增加、减少、清空、覆盖、阈值检查。 | 叠层、消层、读盾、清空气势。 | 对象、状态、计数器名、数量。 |
| 状态转化 | 读取旧状态 / 层数并生成新状态 / 层数。 | 水蚀转寒霜、寒霜转冻结。 | 来源状态、目标状态、数量、额外附加。 |
| 目标选择改写 | 替换本次行动的目标选择方式。 | 混乱。 | 合法目标池、是否允许自己、是否受遮挡。 |
| 行动跳过 | 阻止目标本次行动执行。 | 冻结、眩晕、禁锢。 | 跳过次数、移除方式、免疫判定。 |
| 死亡保护 | 气血归零时阻止死亡确认。 | 续命、免死。 | 触发次数、回血量、触发后是否移除。 |
| 后续规则触发 | 当前规则结算后调用下一条规则链。 | 寄生转疗、爆燃、盾爆、破盾反爆。 | 触发范围、读取结果、下一条规则链。 |

## 5. 规则链模板

| 规则链模板 | 结构 |
| --- | --- |
| 普通伤害 | 目标选择 → 执行器 → 气血减少。 |
| 铺状态 | 目标选择 → 可选气血减少 → 附加状态 / 层数。 |
| 持续伤害 | 回合开始 → 气血减少 → 层数 -1 → 归零移除。 |
| 伤害后转疗 | 指定气血减少结果产生后 → 读取实际减少量 → 选择治疗目标 → 气血增加。 |
| 爆燃 | 读取燃烧层数与强度 → 立即结算火属性气血减少 → 清除燃烧。 |
| 水蚀冷却干扰 | 目标释放主动技能后 → 该技能冷却 +1 → 水蚀层数 -1。 |
| 水蚀转寒霜 | 读取水蚀层数 → 增加寒霜 → 处理水蚀 → 检查寒霜 5 层冻结。 |
| 护盾承伤 | 受到伤害 → 护盾值优先扣除 → 剩余伤害扣气血 → 护盾归零触发。 |
| 盾爆 | 选择读取对象 → 读取护盾值 → 按配置消耗或保留 → 对目标造成伤害。 |
| 混乱改写 | 释放需选目标行动前 → 替换目标选择器 → 原技能继续执行。 |
| 净化 | 选择友方 / 自身 → 移除 1 个可净化状态实例 → 多层整组移除。 |
| 续命 | 气血归零 → 阻止死亡确认 → 气血增加 → 移除续命。 |
| 结界 | 挂载战场 / 阵营 → 指定时机触发规则链 → 计数归零移除。 |

## 6. 触发范围限制

| 限制项 | 用途 | 默认规则 |
| --- | --- | --- |
| 响应结果 | 规定触发器响应哪类结算结果。 | 未写明时，只响应技能 / 状态自己指定的结果。 |
| 响应对象 | 规定谁的行动、受击、伤害、治疗会触发。 | 未写明时，只响应状态所在对象或施法者相关事件。 |
| 响应范围 | 直接伤害、持续伤害、治疗、护盾变化、状态变化等。 | 未写明时，不扩大到其他类型。 |
| 连锁限制 | 防止反弹、吸血、爆燃、法宝追击互相无限触发。 | 默认不允许无限连锁；特殊技能必须写明。 |

## 10. 新技能扩展检查表

| 检查项 | 要求 |
| --- | --- |
| 1. 写规则链 | 先把技能写成“触发 → 目标 → 执行 → 效果 / 数值变动 → 计数器变化”。 |
| 2. 查目标选择器 | 现有选择器能表达就直接引用。 |
| 3. 查执行器 | 单次、多段、随机次数、每次重选等先用已有执行器。 |
| 4. 查状态总表 | 已有状态能表达就引用状态，不在技能里重复解释。 |
| 5. 查效果类型 | 气血变化、护盾变化、数值修正、计数器变化、状态转化等先用基础效果。 |
| 6. 区分系统与显示 | 系统层写真实变动；玩家层写减速、增伤、续命、伤害等表现。 |
| 7. 新增模块 | 只有旧模块表达不了时，才新增一个通用模块。 |
| 8. 技能表写法 | 技能表只写模块名、状态名、目标、执行、冷却和必要参数。 |

## 11. 禁止事项

| 禁止事项 | 原因 |
| --- | --- |
| 禁止每个技能单独写一套完整规则 | 会导致脚本臃肿、重复和难维护。 |
| 禁止把状态细节散落在技能说明里 | 状态统一进状态总表，技能只引用。 |
| 禁止重复解释同一概念 | 人看累，AI 也更难抓结构。 |
| 禁止把系统语言和玩家语言混成一列 | 底层规则与 UI 描述容易互相污染。 |
| 禁止把数值表和机制表混在一起 | 倍率、命中、蓝耗等后续交给独立数据表。 |
| 禁止为了一个技能新增专属模块 | 除非它能变成未来可复用的通用模块。 |
